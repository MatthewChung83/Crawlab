# 改進版法院拍賣爬蟲

這是法院拍賣爬蟲的改進版本，移除了 Scrapy 框架，使用 requests 進行網路爬取，並改善了 PDF 解析精準度。

## 主要改進

### 1. 架構改進
- ✅ **移除 Scrapy 依賴**：使用 `requests` 替代 Scrapy 框架
- ✅ **簡化程式結構**：單一主程式檔案，易於維護
- ✅ **更好的錯誤處理**：完整的例外處理和日誌記錄

### 2. PDF 解析改進
- ✅ **文字標準化**：全形轉半形字元處理
- ✅ **符號清理**：移除無用的標點符號和空白字元
- ✅ **所有權人解析改進**：更準確的姓名提取邏輯
- ✅ **地號資訊改進**：更精確的地號和地段解析
- ✅ **Unicode 正規化**：使用 NFKC 標準化處理

### 3. 資料處理改進
- ✅ **移除 CSV 輸出**：只保留資料庫儲存
- ✅ **重複資料檢查**：透過資料庫查詢避免重複處理
- ✅ **批次處理優化**：減少資料庫連線次數

## 檔案結構

```
├── court_auction_crawler.py    # 主爬蟲程式
├── improved_pdf_reader.py      # 改進版 PDF 解析器
├── config.py                   # 配置檔案
├── requirements_new.txt        # Python 套件需求
└── README_improved.md          # 使用說明
```

## 安裝說明

### 1. Python 環境
需要 Python 3.7 或更高版本

### 2. 安裝相依套件
```bash
pip install -r requirements_new.txt
```

### 3. 資料庫設定
確保可以連接到指定的 SQL Server 資料庫 (在 `config.py` 中設定)

## 使用方法

### 基本使用
```bash
# 爬取今天開始的資料
python court_auction_crawler.py

# 指定開始日期 (YYYYMMDD 格式)
python court_auction_crawler.py --start_date 20240301
```

### 配置選項

在 `config.py` 中可以修改：

- **資料庫連線設定**
- **爬取延遲時間**
- **請求超時設定**
- **輸出目錄路徑**

## 功能特點

### 資料流程
1. **查詢現有資料**：從資料庫檢查已處理的 PDF 檔案
2. **爬取拍賣清單**：使用 requests 獲取法院拍賣資料
3. **下載 PDF 文件**：下載拍賣公告 PDF
4. **解析 PDF 內容**：使用改進版解析器提取所有權人和地號
5. **儲存到資料庫**：將處理結果存入 SQL Server

### PDF 解析改進
- **全形字元處理**：自動轉換全形標點符號
- **文字清理**：移除多餘空白和控制字元
- **所有權人提取**：智慧識別多種分隔符號
- **地號解析**：處理複雜的地號格式

### 錯誤處理
- **網路重試機制**：自動重試失敗的請求
- **詳細日誌記錄**：記錄所有操作和錯誤
- **異常恢復**：單筆資料失敗不影響整體處理

## 輸出資料表

### wbt_court_auction_tb (拍賣基本資訊)
- `rowid`: 唯一識別碼
- `court`: 法院名稱
- `number`: 案件編號
- `date`: 拍賣日期
- `address`: 拍賣標的地址
- `reserve`: 底價
- `handover`: 點交情況
- `vacancy`: 空屋情況
- 等等...

### auction_info_tb (PDF 解析資訊)
- `rowid`: 唯一識別碼
- `owner`: 所有權人
- `parcel`: 地號
- `area`: 面積
- `remark`: 備註
- `referi`: 關聯到拍賣基本資訊的ID
- 等等...

## 效能優化

- **批次處理**：減少資料庫連線次數
- **智慧重複檢查**：避免重複下載和處理
- **可調整延遲**：避免對目標網站造成過度負載
- **記憶體優化**：處理大量 PDF 時的記憶體管理

## 日誌記錄

程式會產生詳細的日誌記錄：
- **INFO 層級**：主要進度和統計資訊
- **DEBUG 層級**：詳細的處理資訊
- **ERROR 層級**：錯誤和異常情況

## 注意事項

1. **網路穩定性**：確保網路連線穩定，PDF 下載需要較多時間
2. **資料庫權限**：確保資料庫使用者有讀寫權限
3. **硬碟空間**：PDF 檔案會暫存到本地，需要足夠空間
4. **爬取頻率**：建議合理設定請求間隔，避免對目標網站造成負擔

## 問題排除

### 常見問題
1. **PDF 解析失敗**：檢查 pdfplumber 套件版本
2. **資料庫連線失敗**：檢查網路和認證資訊
3. **記憶體不足**：可考慮分批處理或增加系統記憶體

### 日誌檔案
程式會產生 `court_auction.log` 檔案，記錄詳細的執行資訊，有助於問題診斷。
